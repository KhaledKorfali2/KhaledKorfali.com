WebSocket isn't a rival to TCP — it's built on top of it. A WebSocket connection starts as a normal HTTP request, asks to "upgrade," and then keeps that same TCP connection open indefinitely so both sides can push messages without re-handshaking every time. UDP skips the handshake and the guarantees entirely, which is exactly why video calls and game state use it — a late packet is more useless than a missing one. QUIC takes a third path: it runs over UDP, but re-implements reliability and ordering itself, per-stream, with encryption baked in from the very first packet — so one lost packet on one stream no longer stalls every other stream sharing the connection. IPv4 sits underneath all of them, responsible only for getting a packet from one address to another, hop by hop, with no promise it will arrive at all.
